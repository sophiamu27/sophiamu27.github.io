Decision Tree

David Zimmermann

History:

There are many variations of the decision tree but the first was invented in 1963 at the University of Wisconsinâ€“Madison in an AID project

How it Works:

A decision tree is a type of supervised machine learning used to categorize or make predictions based on how a previous set of questions were answered. The model is a form of supervised learning, meaning that the model is trained and tested on a set of data that contains the desired categorization.

Pros and Cons:

Pros:

  • It requires less time to get the data prepared for it.
  • The data doesn't have to be normalized
  • No scaling is needed for a decision tree
  • Easy to explain
    •

    Cons:

  • A small change in data can affect it on a large scale
  • The decision tree can give too complex answers
  • The tree can take a long amount of time to calculate
  • The Decision tree does not work very well with regression and can often over fit
    •

    Hyper Parameters:

    For the Hyper Parameters there were many parmeters but the most important on was random state. Random state provided the values for he model

    Outcome:

    The values we gathered from the Decision Tree were:
  • Train value- 1
  • Test value - 0.9814198867197563
  • MSE - 4.71875
    • As you can see these out comes might be desirable but the MSE shows that the Decision Tree was over fitting which would ot work for our data

    Support Vector Regression

    Katherine Xie

    History:

    This machine learning model was invented by Vladimir N. Vapnik and Alexey Ya. Chervonenkis in 1963

    How it Works:

    SVM is a supervised machine learning algorithm that can be used for classification or regression. It finds a hyperplane that divides a data set into different classes. The goal for this method would be to choose a plane with the greatest margin between the hyperplane and a data point.

    Pros and Cons:

    Pros:

  • Thrives in High Dimension
  • Support Vector Machines are all about choosing the right kernel with the right parameters and this can provide lots of flexibility
  • Fast Prediction

    • Cons:

  • Not good for big data sets because training will take a long time
  • Less effective on messy data sets
  • Not the most efficient algorithms and it can be quite costly computationally to train them
  • Can be hard to interpret
  • Have to scale data

    • Hyper Parameters:

    We found that the best hyper parameters were: SVR(kernel='rbf', gamma='scale', tol=0.001, C=100)
    Scores:

    Outcome:

    After analyzing the results, we found that SVR produced fairly accurate predictions but had a high mean squared error, which means the model has large possibility for error.

    • Train Score- 0.9999799773106288
    • Test Score- 0.9217180265305381
    • Mean Squared Error- 117.1152028382671

    Random Forest

    Sophia Mu

    History:

    Made by Tin Kam Ho in 1995

    How it Works:

    Random forest is a machine learning model. This model uses many decision trees to put in the input and calculate the output based on the mean or mode of the predictions made by the decision trees.

    Pros and Cons:

    Pros:

  • can be used for classification and regression
  • works well with non-linear data
  • works efficiently with large datasets
  • has a higher level of accuracy compared to decision tree
  • has many parameters

    • Cons:

  • does not work well with linear data
  • is known to overfit
  • slow training

    • Hyper Parameters:

    • n_estimators
    • max_features
    • max_depth
    • min_samples_split
    • min_samples_leaf
    • random_state
    • n_jobs
    • oob_score

    Outcome:

    • Train data after adding on parameters was 0.9895121580560061
    • Test data after adding on parameters was 0.9745539111597259
    • We did not end up using this model because other models were more accurate in the train and test results.

    Lasso Regression

    Amir Ray

    LASSO:

  • Least
  • Absolute
  • Shrinkage
  • Selection
  • Operator

    History:

    Lasso regression was developed independently in geophysics literature in 1986. Statistician Robert Tibshirani independently rediscovered and popularised it in 1996.

    How it Works:

    Lasso regression is a type of linear regression that is more accurate. This variant of linear regression uses shrinkage, which helps the results become more accurate. This type of regression specializes in models showing high levels of multicollinearity. Lasso regression is also a regularization technique. This means that the data will most likely not overfitting. This linear regression technique has proven itself to be reliable and accurate.

    Pros and Cons:

    Pros:

  • Avoids overfitting
  • Can do feature selection
  • Is fast when fitting data

    Cons:

  • The model selected by lasso may be unstable
  • Lasso may randomly select things
  • Lasso is automatic which can be bad

    Hyper Parameters:

    We found that certain parameters increase our accuracy and MSE. Certain combinations proved to be reliable and accurate. The best hyperparameters for lasso are:

    Lasso(tol=0.000001, fit_intercept=False, precompute=False, max_iter=100000, warm_start=True, positive=False, copy_X=True, alpha=0.001 )

    Outcome:

  • Accuracy(Train): 0.999999902536181
  • Accuracy(Test): 0.9999998915243217
  • MSE: 0.7180257402316035

    • Linear Regression

    Nne Ezi

    The earliest form of linear regression was the least square method, which was published in 1805 by Legendre.

    linear regression is a machine learning model that helps predict a linear relationship between input variables and single output variables. It is the process of finding the best fit line for a data points on a plot. <3>

    Pros and Cons:

    Pros:

  • It performs well for linear separable data
  • It is easy to implement, and efficient at training
  • It handles overfitting well

    • Cons:

  • It doesn't consider outliers.
  • It assumes a linear relationship between the independent and dependent variables

    • Hyper Parameters:

    While using the machine learning of linear regression, the training data accuracy was 1.0, and for the testing data the accuracy was 0.99. Even when the parameter were change the accuracy level was consistent.

    Outcome:

    All the accuracy scores for linear regression came out really high, near perfect, which we found might be too good to be true. It also had very low chance for error.

  • Train Accuracy: 1.0
  • Test Accuracy: 0.99
  • Mean Squared Error: 0.0007

    About

    Our Story

    • What is Cost of Living

      Cost of living is the amount of money it would cost you to live in a specific area or city. Cost of living, itself, may sound like it only refers to where you are actually staying like a house or an apartment, but it is ,in fact, much more. It takes into consideration everything that you could have to spend money on from gas to groceries.

    • Why did we do it

      The reason our team created the Cost of living project is because it would be extremely useful to the world if we do it correctly. We want to give people more information when they are planning to leave and live somewhere else. This project will give people more confidence going into their new life so they do not get surprised by the price of anything.

    • How did we do it

      This website is very easy to follow and will be accessible to anyone who has an internet connection. Just put in a name of a city that you have thought about living and then our database will find your city and give you the numbers.If the city you request is not inn our extensive library of cities our AI will calculate the prices based on the other cities in the area and you will be informed that the price is not certain.

    • Future Inventions

      On top of idea we also are bringing new and special additions to our website. We will have a feature where, if you have not chosen a city or you are just curious,you can input a budget that you have set for yourself and our code will give you all the cities in our price range. You will also get a picture of the city along with an interactive map showing the surrounding area.
